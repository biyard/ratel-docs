## ğŸ“ Frontmatter Guidelines

### Required Fields
- `title`: Page title (shown in browser tab and sidebar)
- `description`: Brief description for SEO and previews
- `sidebar_position`: Number to control order in sidebar

### Optional Fields
- `slug`: Custom URL path (e.g., `/` for homepage)
- `tags`: Array of tags for categorization
- `keywords`: Array for SEO

### Example

```yaml
---
title: Creating Your First Space
description: Step-by-step guide to creating a Space on Ratel
sidebar_position: 3
tags: [spaces, getting-started]
keywords: [ratel space, create space, governance]
---
```

## ğŸ“š Documentation Content Guidelines

When filling in content:

1. **Be Concise**: Users want quick answers
2. **Use Examples**: Show, don't just tell
3. **Include Screenshots**: Visual guides are helpful (store in `static/img/` or `docs/`)
4. **Link Related Pages**: Use relative links like `[Link](./other-page.md)`
5. **Maintain Consistency**: Follow the same structure across pages
6. **Test Locally**: Always preview with `npm start` before committing
