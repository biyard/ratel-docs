# Screenshot Capture Guide

This document outlines all screenshots needed for the Ratel documentation User Guide section.

## Screenshot Specifications

- **Format**: PNG
- **Resolution**: At least 1920x1080 for desktop, 375x812 for mobile
- **Quality**: High quality, no compression artifacts
- **Content**: 
  - Hide personal information
  - Use example data that's appropriate for documentation
  - Ensure UI is in a clean, professional state
  - Capture at 100% or 200% zoom (retina)
- **File Naming**: Use kebab-case, descriptive names
- **Annotations**: Add red boxes or arrows only when needed to highlight specific features

## Additional Considerations

### Responsive Design
- Capture both desktop and mobile views where UI differs significantly
- Use suffixes: `-desktop.png`, `-mobile.png`

### Dark Mode
- If the platform supports dark mode, capture both themes
- Use suffixes: `-light.png`, `-dark.png`

### Localization
- Capture screenshots in English (primary)
- Korean screenshots can be added later in `/ko/img/screenshots/` if needed

### Annotations
- Use red color (#FF0000) for highlights
- Arrow thickness: 3px
- Box border: 2px solid
- Keep annotations minimal

## Screenshot Workflow

1. **Capture**: Take screenshot at specified resolution
2. **Crop**: Remove unnecessary browser chrome (keep only relevant UI)
3. **Annotate**: Add highlights if needed
4. **Optimize**: Compress to reasonable file size (<500KB ideally)
5. **Name**: Use descriptive kebab-case filename
6. **Place**: Save to appropriate directory
7. **Reference**: Update documentation with correct path

## Tools Recommended

- **macOS**: Cmd+Shift+4 (built-in), or CleanShot X
- **Windows**: Snipping Tool, ShareX
- **Linux**: Spectacle, Flameshot
- **Annotation**: Skitch, Snagit, or built-in tools
- **Optimization**: TinyPNG, ImageOptim
