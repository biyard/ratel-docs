# 보안 취약점 신고

Ratel의 보안을 유지하는 데 도움을 주시는 보안 연구 커뮤니티에 감사드립니다.

## 책임 있는 공개

보안 취약점을 발견하시면 책임감 있게 신고해 주세요:

### 신고 방법

1. **이메일**: [security@biyard.co](mailto:security@biyard.co)
2. **제목**: "보안 취약점 신고"
3. **포함 사항**:
   - 취약점 설명
   - 재현 단계
   - 잠재적 영향
   - 제안된 해결 방법(있는 경우)
   - 연락처 정보

### 당사의 약속

- **확인**: 48시간 이내
- **초기 평가**: 영업일 기준 5일 이내
- **해결 일정**: 심각도에 따라
  - 긴급: 7일
  - 높음: 14일
  - 중간: 30일
  - 낮음: 90일
- **공개 공지**: 수정 배포 후 조율된 공개

## 범위

### 범위 내
✅ 인증 및 권한 부여 결함
✅ 데이터 누출 또는 노출
✅ SQL 인젝션 및 XSS 취약점
✅ 스마트 계약 취약점
✅ 암호화 약점
✅ 서버 측 코드 실행

### 범위 외
❌ 사회 공학 또는 피싱
❌ 인프라에 대한 물리적 공격
❌ 서비스 거부(DoS/DDoS)
❌ 스팸 또는 콘텐츠 문제
❌ 물리적 액세스가 필요한 문제

## 가이드라인

**해야 할 일**:
- 상세한 취약점 보고서 제공
- 수정을 위한 합리적인 시간 허용
- 공개까지 기밀 유지
- 자신의 계정에서만 테스트

**하지 말아야 할 일**:
- 다른 사용자의 데이터 액세스 또는 수정
- DoS/DDoS 공격 수행
- 개념 증명을 넘어 취약점 악용
- 당사가 문제를 해결하기 전에 공개적으로 공개

## 인정

당사는 책임 있는 공개 기여자를 인정하는 보안 연구원 명예의 전당을 유지합니다.

## 버그 바운티 프로그램

긴급 취약점에 대해 최대 $15,000의 보상을 제공하는 버그 바운티 프로그램을 2025년 2분기에 시작합니다.

## 법적 안전 항구

다음과 같은 보안 연구원에 대해서는 법적 조치를 취하지 않습니다:
- 책임 있는 공개 관행을 따르는 경우
- 선의로 행동하는 경우
- 사용자 개인정보를 침해하지 않는 경우
- 서비스를 방해하지 않는 경우

Ratel의 보안을 유지하는 데 도움을 주셔서 감사합니다!
