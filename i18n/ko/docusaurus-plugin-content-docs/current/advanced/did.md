# 탈중앙화 신원 (DID)

탈중앙화 신원 및 Ratel에서 사용하는 방법을 알아보세요.

:::info 베타 기능
DID 기능은 현재 **오라클 검증 시스템**의 일부로 베타 버전입니다(2025년 1분기). 전체 DID 구현은 2025년 2분기 예정입니다.
:::

## DID란 무엇인가요?

**탈중앙화 신원(DID)**은 검증 가능하고 자기 주권적인 디지털 신원을 가능하게 하는 새로운 유형의 식별자입니다. 중앙 권한이 제어하는 기존 신원과 달리, DID는 귀하에게 신원 자격 증명에 대한 완전한 소유권과 제어권을 부여합니다.

### 핵심 원칙

**자기 주권**
- 신원을 소유합니다
- 중앙 권한이 제어하지 않습니다
- 공유할 내용을 결정합니다

**검증 가능**
- 클레임을 암호학적으로 검증 가능
- 커뮤니티 기반 증명
- 무신뢰 검증

**이식 가능**
- 플랫폼 간 사용
- Ratel에 종속되지 않음
- 상호 운용 가능한 표준

## Ratel의 DID 사용

### 오라클 기반 검증

Ratel의 DID 시스템은 **오라클**을 사용합니다 - 커뮤니티 구성원이 신원 속성을 검증하기 위해 투표하는 다자간 검증 메커니즘입니다.

**프로세스:**
1. 검증 요청 제출
2. 클레임에 대한 증거 제공
3. 오라클 투표자가 증거 검토
4. 커뮤니티 투표로 승인/거부
5. 검증된 속성이 DID에 추가

### 확장 가능한 속성

정부 검증에 제한된 기존 KYC와 달리, Ratel의 오라클은 다양한 속성 검증을 가능하게 합니다:

#### 정부 속성
- 연령 검증 (18+, 21+)
- 거주 국가
- 정부 발급 ID 검증
- 전문 라이선스

#### 전문 자격
- 산업 전문성
- 기술 및 인증
- 업무 경험
- 교육 배경
- 포트폴리오 검증

#### 평판 지표
- 커뮤니티 지위
- 기여 이력
- 전문 분야
- 신뢰 점수

#### 소셜 증명
- 조직 멤버십
- 이벤트 참석
- 커뮤니티 소속
- 업적 배지

#### 사용자 지정 속성
- 커뮤니티가 가치 있게 여기는 모든 속성
- 도메인별 자격 증명
- 특별 인정
- 역할 기반 속성

## DID 설정하기

### 1단계: DID 문서 생성

1. 설정 → **신원**으로 이동
2. "DID 생성" 클릭
3. DID가 생성됩니다:
   ```
   did:ratel:user_abc123
   ```

### 2단계: 기본 프로필 추가

DID 문서에는 다음이 포함됩니다:

```json
{
  "@context": "https://www.w3.org/ns/did/v1",
  "id": "did:ratel:user_abc123",
  "authentication": [{
    "id": "did:ratel:user_abc123#key-1",
    "type": "Ed25519VerificationKey2020",
    "controller": "did:ratel:user_abc123",
    "publicKeyMultibase": "z6Mk..."
  }],
  "service": [{
    "id": "did:ratel:user_abc123#profile",
    "type": "Profile",
    "serviceEndpoint": "https://ratel.foundation/users/alice"
  }]
}
```

### 3단계: 검증 요청

속성에 대한 검증 요청 제출:

1. 설정 → **신원** → **검증**
2. "검증 요청" 클릭
3. 속성 유형 선택
4. 증거 제공
5. 오라클에 제출

## 검증 유형

### 1. 이메일 검증

**필요한 증거:**
- 검증된 이메일 주소

**프로세스:**
- 자동 검증
- 투표 불필요
- 즉시 승인

**혜택:**
- 비밀번호 복구
- 알림 기본 설정
- 기본 신뢰 신호

### 2. 정부 ID (KYC)

:::caution 출시 예정
정부 ID 검증은 2025년 2분기에 제3자 KYC 제공업체를 통해 예정되어 있습니다.
:::

**필요한 증거:**
- 정부 발급 ID 사진
- 활성 검사를 위한 셀카
- 주소 증명 (선택 사항)

**프로세스:**
- 제3자 KYC 검증
- 프라이버시 보호 (영지식 증명)
- 예/아니오 결과만 온체인에 저장

**혜택:**
- 연령 제한 스페이스 액세스
- 더 높은 평판 점수
- 규제 활동에 대한 규정 준수

### 3. 전문 기술

**필요한 증거:**
- 포트폴리오 링크 (GitHub, Behance 등)
- 인증서
- 업무 이력
- 프로젝트

**프로세스:**
- 커뮤니티 오라클 투표
- 해당 분야 전문가가 증거 검토
- 7일 투표 기간
- 67% 승인 임계값

**예시:**

```markdown
**속성:** Rust 개발자
**증거:**
- GitHub: github.com/alice (20개 이상의 Rust 프로젝트)
- Crates.io: 5개의 공개 크레이트
- 기여: Tokio, Actix-web
- 인증: Rust 프로그래밍 인증
```

### 4. 커뮤니티 평판

**필요한 증거:**
- 기여 이력
- 커뮤니티 피드백
- 품질 지표

**프로세스:**
- 자동 계산
- 플랫폼 활동 기반
- 월별 업데이트

**지표:**
- 게시물 품질 점수
- 유용한 댓글 비율
- 숙의 참여
- 중재 기록

### 5. 조직 멤버십

**필요한 증거:**
- 공식 멤버십 증명
- 조직 승인
- 공개 기록

**프로세스:**
- 조직 관리자 검증
- 또는 커뮤니티 오라클 투표
- 연간 갱신 가능

**예시:**
- 회사 직원
- DAO 멤버
- 비영리 자원봉사자
- 학술 기관

## DID 사용하기

### Ratel에서

**프로필 표시**
- 프로필에 검증 배지
- 속성 배지 표시
- 신뢰 점수 표시

**액세스 제어**
- 검증된 사용자만 가입 가능한 스페이스 가입
- 게이트 숙의 참여
- 프리미엄 기능 액세스

**평판**
- 더 높은 초기 신뢰
- 더 나은 콘텐츠 가시성
- 중재 권한

### 크로스 플랫폼

**DID 내보내기**

1. 설정 → **신원** → **내보내기**
2. DID 문서 다운로드
3. 다른 플랫폼에서 사용

**호환 플랫폼:**
- 기타 DID 지원 dApp
- Web3 애플리케이션
- 탈중앙화 소셜 네트워크

### 검증 가능한 자격 증명

서명된 자격 증명 요청:

```json
{
  "@context": "https://www.w3.org/2018/credentials/v1",
  "type": ["VerifiableCredential", "SkillCredential"],
  "issuer": "did:ratel:oracle",
  "issuanceDate": "2025-03-20T10:00:00Z",
  "credentialSubject": {
    "id": "did:ratel:user_abc123",
    "skill": "Rust Developer",
    "evidenceUrl": "https://ratel.foundation/oracle/verification_xyz789"
  },
  "proof": {
    "type": "Ed25519Signature2020",
    "created": "2025-03-20T10:00:00Z",
    "verificationMethod": "did:ratel:oracle#key-1",
    "proofPurpose": "assertionMethod",
    "proofValue": "z5s..."
  }
}
```

## 프라이버시 및 보안

### 프라이버시 제어

**공유할 내용 선택**
- 속성별 가시성 제어
- 공개, 연결만 또는 비공개
- 선택적 공개

**영지식 증명**
- 생년월일을 밝히지 않고 연령 증명
- 정확한 주소 없이 위치 증명
- 세부 정보 없이 자격 증명

### 보안 기능

**키 관리**
- 개인 키 자체 보관
- 하드웨어 지갑 지원
- 소셜 복구 옵션

**취소**
- 언제든지 자격 증명 취소
- 오래된 속성 업데이트
- 손상된 검증 제거

**감사 추적**
- 모든 검증 이력 보기
- 누가 무엇을 검증했는지 추적
- 투명한 투표 기록

## 오라클 투표

### 검증자 되기

요구사항:
- RATEL 토큰 보유자 (최소 100개)
- 계정 연령 >30일
- 좋은 평판 점수
- 관련 전문 지식 (일부 속성의 경우)

### 투표 프로세스

1. 전문 분야의 검증 요청 검토
2. 증거를 신중하게 검토
3. 정당화와 함께 승인/거부 투표
4. 정확한 투표에 대한 보상 획득

### 투표 보상

- **정확한 투표**: RATEL 토큰 획득
- **부정확한 투표**: 평판 손실
- **악의적인 투표**: 토큰 슬래싱

## 모범 사례

### 사용자용

✅ **할 일:**
- 포괄적인 증거 제공
- 클레임에서 정직하기
- 자격 증명을 최신 상태로 유지
- 여러 검증 유형 사용
- 프라이버시 설정 존중

❌ **하지 말아야 할 일:**
- 허위 클레임 제출
- 개인 키 공유
- 개인 정보 과다 공유
- 한 자격 증명에만 의존
- 취소 필요성 무시

### 검증자용

✅ **할 일:**
- 증거를 철저히 검토
- 전문 분야 내에서 투표
- 건설적인 피드백 제공
- 공정하고 편견 없이
- 의심스러운 요청 보고

❌ **하지 말아야 할 일:**
- 검토 없이 투표
- 무조건 승인
- 개인적 편견에 따른 투표
- 투표 이유 공개적으로 공유
- 투표에 대한 뇌물 수수

## 자주 묻는 질문

**Q: Ratel을 사용하려면 DID가 필요한가요?**
아니요, DID는 완전히 선택 사항입니다. 경험을 향상시키지만 필수는 아닙니다.

**Q: 어떤 데이터가 온체인에 저장되나요?**
검증 결과(예/아니오)와 해시만 저장됩니다. 개인 데이터는 온체인에 저장되지 않습니다.

**Q: DID를 삭제할 수 있나요?**
예, 언제든지 DID를 취소할 수 있습니다. 온체인 기록은 삭제되지 않지만 취소됨으로 표시됩니다.

**Q: 검증은 얼마나 오래 지속되나요?**
- 이메일: 이메일이 변경될 때까지
- 정부 ID: 2년
- 기술: 1년 (갱신 가능)
- 평판: 월별 업데이트
- 조직: 멤버십이 끝날 때까지

**Q: 검증 요청이 거부되면 어떻게 되나요?**
더 나은 증거로 다시 제출하거나 다른 오라클 풀에 이의를 제기할 수 있습니다.

**Q: 검증 비용은 얼마인가요?**
- 이메일: 무료
- 정부 ID: ~$5 (KYC 제공업체 비용 포함)
- 오라클 검증: 소량의 RATEL 토큰 수수료 (스팸 방지)
